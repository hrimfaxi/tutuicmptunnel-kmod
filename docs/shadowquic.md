# shadowquic

[English](./shadowquic.md) | [简体中文](./shadowquic_zh-CN.md)
---


`shadowquic` is a proxy tool built on **Rust** and **quinn (QUIC)**. Its overall design is similar to Hysteria: it uses **UDP/QUIC** transport to achieve higher throughput and better performance on unstable networks.

> Version requirement: use **v0.3.3 or newer** so you can disable **GSO** (required for `tutuicmptunnel-kmod`).

---

## Features & Benefits

1. **Fast, with lower resource usage**
   In my test environment (mipsle, Xiaomi R3G), `shadowquic` uses less memory than `hysteria` (about **1/3**), and under high load its CPU usage is about **50%**.

2. **JLS (certificate-free deployment)**
   `shadowquic` uses JLS, a TLS 1.3–style handshake/obfuscation and authentication scheme that mainly relies on shared credentials instead of the traditional certificate system. In practice, this usually means you don’t need ACME-issued certificates or a self-managed certificate chain like you typically do with `hysteria`.

   - The client-side `server-name` is used to match the handshake “appearance”/fingerprint.
   - It usually needs to be consistent with the server’s `jls_upstream`.

3. **Rust memory safety**
   Rust significantly reduces common memory-safety issues found in traditional C/C++ implementations.

## Installation & Configuration

### Server

Install and start the service:

```bash
curl -L https://raw.githubusercontent.com/spongebob888/shadowquic/main/scripts/linux_install.sh | bash
systemctl daemon-reload
systemctl enable --now shadowquic.service
```

#### Disable GSO for tutuicmptunnel-kmod

Edit `/etc/shadowquic/server.yaml` and add (or ensure) the following under `inbound`:

```yaml
inbound:
  ...
  gso: false
```

Then restart the service:

```sh
systemctl restart shadowquic.service
```

---

### Client

Fill in the username/password generated by the server script into `client.yaml`, and **disable GSO** as well:

```yaml
inbound:
  type: socks
  bind-addr: "127.0.0.1:20808"

outbound:
  type: shadowquic
  addr: "1.2.3.4:1443"
  username: "username"           # Username
  password: "password"           # Password
  server-name: "cloudflare.com"  # Usually should match server jls_upstream (fingerprint alignment)
  alpn: ["h3"]
  initial-mtu: 1000
  congestion-control: bbr
  zero-rtt: true
  gso: false                     # for tutuicmptunnel-kmod
  over-stream: false             # true: UDP over stream; false: UDP over datagram

log-level: "info"
```

Start the client:

```bash
shadowquic -c client.yaml
```

You can now protect it with `tutuicmptunnel-kmod`. The method is similar to [hysteria](/docs/hysteria.md)—just change the port to **1443**.
